# Wavly 🖐️ — AI-Powered Gesture Interface

Touchless computer control using real-time hand tracking and gesture recognition.

---

## Project Structure

```
wavly/
├── main.py                    # Entry point — starts all threads + UI
│
├── core/
│   ├── camera_thread.py       # Thread 1: Webcam → MediaPipe → Gesture Queue
│   ├── action_thread.py       # Thread 2: Gesture Queue → PyAutoGUI actions
│   └── gesture_queue.py       # Thread-safe event bridge
│
├── gestures/
│   ├── classifier.py          # Rule-based + ML gesture classifier
│   ├── landmark_utils.py      # Feature engineering from MediaPipe landmarks
│   └── trainer.py             # Interactive training tool for custom gestures
│
├── ui/
│   └── overlay.py             # PyQt6 transparent HUD overlay
│
├── config/
│   └── settings.py            # All tuneable parameters
│
├── models/
│   └── gesture_model.pkl      # Trained model (generated by trainer.py)
│
└── requirements.txt
```

---

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

> Python 3.10+ required. Virtual environment recommended.

### 2. Run immediately (rule-based mode — no training needed)

```bash
python main.py
```

The system starts in **rule-based mode** using finger state logic.
It works out of the box — no training required for core gestures.

### 3. Train a custom ML model (optional but recommended)

```bash
python gestures/trainer.py
```

Follow the prompts to record ~200 samples per gesture.
After training, the model is saved to `models/gesture_model.pkl`
and loaded automatically on next run.

## Default Gestures

| Gesture | Action |
|---------|--------|
| ☝️ Index finger pointing | Move cursor |
| 🤌 Thumb + index pinch | Left click |
| 🤌 Thumb + middle pinch | Right click |
| ✌️ Two fingers up, raised | Scroll up |
| ✌️ Two fingers up, lowered | Scroll down |
| ✊ Closed fist | Start drag |
| 🖐️ Open palm | Stop / release drag |
| 🤏 Spread thumb + index | Zoom in (Ctrl++) |
| 🤏 Close thumb + index | Zoom out (Ctrl+-) |

## Configuration

Edit `config/settings.py` to tune:

| Parameter | Default | Effect |
|-----------|---------|--------|
| `hold_frames` | 8 | Frames before gesture fires (anti-jitter) |
| `cursor_smoothing` | 0.35 | 0.1=smooth, 1.0=raw |
| `scroll_speed` | 3 | PyAutoGUI scroll units |
| `show_debug_window` | False | Show OpenCV landmark preview |
| `ml_confidence_threshold` | 0.65 | Min ML confidence before fallback |

## Architecture

```
┌─────────────┐    ┌──────────────────────────────────────────────────┐
│  Webcam     │───▶│  CameraThread (Thread 1)                         │
└─────────────┘    │  OpenCV → MediaPipe → LandmarkUtils → Classifier │
                   │  + Debounce filter                                │
                   └──────────────────────┬───────────────────────────┘
                                          │ GestureEvent
                                          ▼
                              ┌─────────────────────┐
                              │   GestureQueue      │◀──── UI polls (peek)
                              └──────────┬──────────┘
                                         │
                   ┌─────────────────────▼───────────────────────────┐
                   │  ActionThread (Thread 2)                         │
                   │  cursor_move → PyAutoGUI.moveTo (smoothed)       │
                   │  click/scroll/drag → PyAutoGUI actions           │
                   └─────────────────────────────────────────────────┘

                   ┌─────────────────────────────────────────────────┐
                   │  WavlyOverlay (PyQt6, Thread 3 via QTimer)       │
                   │  Transparent HUD — gesture name + confidence     │
                   └─────────────────────────────────────────────────┘
```

## Troubleshooting

**Cursor is jittery**
→ Increase `cursor_smoothing` value (try 0.2)
→ Improve lighting — avoid backlit setups

**Gestures fire accidentally**
→ Increase `hold_frames` (try 12–15)
→ Train a custom ML model for your hand

**No hand detected**
→ Set `show_debug_window = True` to see what MediaPipe sees
→ Check lighting — MediaPipe struggles below ~100 lux

**Low ML accuracy after training**
→ Record more samples (aim for 300+ per gesture)
→ Record in the same lighting conditions you'll use Wavly in
